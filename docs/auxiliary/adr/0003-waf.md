# ADR 0003 - WAFv2 enabled at v1

## Status
Accepted

## Context
PRD listed WAF as optional. With a public sign-up form, automated abuse is
expected.

## Decision
Attach a WAFv2 web ACL to the ALB with managed rule sets:
- `AWSManagedRulesCommonRuleSet`
- `AWSManagedRulesKnownBadInputsRuleSet`
- `AWSManagedRulesSQLiRuleSet`
- per-IP rate limit (2000 req / 5m)

## Consequences
- Modest extra cost for managed rules + WCU usage.
- False positives possible against admin CSV export with unusual content.
  Mitigated by visibility config (sampled requests) so we can tune rule
  groups (override actions to count) before enforcing.
